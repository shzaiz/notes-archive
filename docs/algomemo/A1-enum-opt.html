1.从抽象算法到具体算法
在设计一个具体的算法的时候, 怎么考量和写作啊?

算法正确性的证明. 通常是使用一系列陈述分解现在一定满足的状态, 比如

  • 在当前行, 变量X一定不为空

  • 在循环的过程中, 性质Y一定满足

它们要么可以通过数学归纳法或者结构归纳法方法证明(类似与Rocq定理证明器中的induction策略), 要么可以通过程序的语义说明. 在分析问题的时候可以使用分类讨论的方法. 实际中, 可以把这些不变式写到对应的assert中, 以便在出现问题的时候及时地退出.

常见评估技巧. 具体每种情况会在后续介绍

  • 理论上: 写为表达式, 然后通过放缩等手段进行估计, 得到上界和下界.

  • 实际上: 写成计算机程序 → 不同的量级运行 → 画图结果

    • 由于真实系统的复杂性(缓存等), 往往有出入.

5.维护一个恰好满足性质的区间

例 5. 2563. 统计公平数对的数目

答案. 排序之后的结果: 两个指针越靠后, 答案越大; 满足单调性相关: 区间越长, 答案越大.

  • 枚举一个数, 决定左右两边的范围

代码演示 

class Solution {
public:
// 0 1 4 4 5 7
// ~   L   R   --> 3
//   ~ L   R   --> 2

// 1 2 5 7 9 [11, 11]
// ~       LR
//   ~     LR
    long long countFairPairs(vector<int>& nums, int lower, int upper) {
        // from smallest to biggest
        sort(nums.begin(), nums.end());
        long long ans = 0;
        int n = nums.size();
        int rLoPtr = n-1, rHiPtr = n-1;
        for(int lptr = 0; lptr < n; lptr++){
            if((lptr > rLoPtr)) {
                rLoPtr = max(rLoPtr, lptr);
                rHiPtr = max(rHiPtr, lptr);
            }
            
            while((rLoPtr > lptr) && 
                                  (nums[lptr] + nums[rLoPtr] >= lower)) 
                rLoPtr--;
            while((rHiPtr > lptr) && 
                                  (nums[lptr] + nums[rHiPtr] > upper)) 
                rHiPtr--;
               
            ans += (rHiPtr - rLoPtr);
        }
        return ans;
    }
};
