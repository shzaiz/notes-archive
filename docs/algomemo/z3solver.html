2.约束条件

想象一下，你来到了一个神奇的城堡，城堡的门上挂着许多谜题。这些谜题只有“对”或“错”的答案，就好像开关只有开和关两种状态。城堡的门只有在所有谜题都回答正确时才能打开。比如，你得到一条消息说:

“如果第一个开关打开，那么第三个开关必须关闭；如果第二个开关关闭，则第四个开关也必须关闭。”

这样的问题，人类总是采用“假设—推导—验证”的方法：

  • 假设1：第一个开关开

    • 根据条件一，则第三个开关必须关。

    • 然后考虑第二个开关：

      • 如果第二个开关关，则根据条件二第四个也必须关；

      • 如果第二个开关开，那么第四个开关可以是任意状态，因为条件二只对“第二个关”的情况有约束。

  • 假设2：第一个开关关

    • 此时条件一不产生限制，第三个开关可以自由选择（开或关）。

    • 依然对第二个开关和第四个开关应用条件二：

      • 如果第二个关，则第四个必须关；

      • 如果第二个开，第四个可以是任意状态。

这种“分别讨论不同情况”的方法，能帮助人们分清哪些状态组合可以行得通。

实际上，在我们的思维逻辑中，有时候还会偷懒：比如一旦确定开关1是开的，那么马上可以得到开关3必须关的结论，无需再考虑开关3的其他可能状态。

诶, 电脑有强大的运行速度。要是能借助电脑来枚举情况就好了...

实际上, 他们的工作并不是粗暴地枚举, 而是基于人类的一些观察

更有针对性地进行尝试, 就像刚刚我们推理的时候偷的懒一样.

from z3 import *

# 定义四个布尔变量，分别代表四个开关

s1 = Bool('s1') # 第一个开关

s2 = Bool('s2') # 第二个开关

s3 = Bool('s3') # 第三个开关

s4 = Bool('s4') # 第四个开关

# 创建求解器实例

solver = Solver()

# 添加约束条件：

# 条件1：如果第一个开关开启（True），则第三个开关必须关闭（False）。

# 用 Z3 中的 Implies 表示：Implies(s1, Not(s3)) 相当于 s1 -> ¬s3

solver.add(Implies(s1, Not(s3)))

# 条件2：如果第二个开关关闭（False），则第四个开关必须关闭（False）。

# 用 Z3 中的 Implies 表示：Implies(Not(s2), Not(s4)) 相当于 ¬s2 -> ¬s4

solver.add(Implies(Not(s2), Not(s4)))

# 检查公式是否可满足

if solver.check() == sat:

# 获取满足条件的模型

m = solver.model()

print("找到一个满足约束的解：")

print(f"s1: {m[s1]}, s2: {m[s2]}, s3: {m[s3]}, s4: {m[s4]}")

else:

print("公式不可满足！")

可以发现，SAT 求解器只关心“对”与“错”的二元问题。它适合用来解决那些只涉及简单逻辑（真、假）的谜题，比如上面那种开关谜题。每个谜题条件的答案都只有“真”或“假”。

3.约束条件 II.

可是上面的约束条件并不能表达诸如“这个数得在0~9之间”

这样的条件呀

没错! 但是我们可以来做一点转换. 我们这里选取一个最简单的

想法...

我们为每个单元格可能的数字定义布尔变量。⁠例如，变量 表示在第 行、第 列的单元格中填入数字 。⁠对于 9×9 的数独，总共有 9×9×9 = 729 个这样的变量。

我明白了, 只要加的变量足够多就可以把它转换为一个可以求解

的问题了! 只需要翻译成对应的约束条件就好了!

但是在现代的SMT Solver中, 你并不需要手工创建这么多变量

有相当大的一部分是由SMT Solver自动完成的.

假设我们有一个的数独, 我们可以把数独的格子建模为 

X = [[ Int("x_%s_%s" % (i+1, j+1)) for j in range(9) ]
 for i in range(9) ]

下面考虑约束条件:

  • 每一个格子必须包含1∼9的一个整数: 

    cells_c  = [ And(1 <= X[i][j], X[i][j] <= 9) 
             for i in range(9) for j in range(9) ]

  • 每个数字必须在一列中仅仅出现1次 

    rows_c   = [ Distinct(X[i]) for i in range(9) ]

  • 每个数字必须在一行中仅仅出现1次 

    cols_c   = [ Distinct([ X[i][j] for i in range(9) ])
            for j in range(9) ]

  • 每个数字必须在的小方格中仅仅出现1次 

    sq_c     = [ Distinct([ X[3*i0 + i][3*j0 + j]
                        for i in range(3) for j in range(3) ])
                       for i0 in range(3) for j0 in range(3) ]

最后可以把这些元素加起来, 表示他们都要被满足: 

sudoku_c = cells_c + rows_c + cols_c + sq_c

这就是我们数独的模型, 下面只要给它一个具体的实例...

这里面0是空的位置 

instance = ((5,3,0,0,7,0,0,0,0),
            (6,0,0,1,9,5,0,0,0),

那些填过的数呢? 是不是也要作为约束放进来?

确实是这样的, 现在就有更多的约束条件了 

instance_c = [ If(instance[i][j] == 0, 
                  True, 
                  X[i][j] == instance[i][j]) 
               for i in range(9) for j in range(9) ]
最后, 就是让这个Solver慢慢搜索啦! 

s = Solver()
s.add(sudoku_c + instance_c)
if s.check() == sat:
    m = s.model()
    r = [ [ m.evaluate(X[i][j]) for j in range(9) ]
          for i in range(9) ]
    print_matrix(r)
else:
    print("failed to solve")
